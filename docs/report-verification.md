# Report Verification

## Endpoints
- `GET /public/report/verify?token=<token>`
- `GET /public/report/verify/receipt?token=<token>`

## Purpose
Validate that an exported report was generated by IFMS, determine its verification assurance level, and provide evidence for auditors/regulators.

## Response

```json
{
  "valid": true,
  "exportType": "reports.overview",
  "format": "pdf",
  "verificationLevel": "ltv",
  "isSigned": true,
  "signingStatus": "signed",
  "signedAt": "2026-02-15T09:31:20.000Z",
  "tsaStatus": "stamped",
  "tsaProvider": "trusted-tsa",
  "pdfaLevel": "PDF/A-2b",
  "signatureProfile": "PAdES-B-LT",
  "generatedAt": "2026-02-15T09:31:00.000Z",
  "sha256": "...",
  "expiresAt": "2026-02-18T09:31:00.000Z",
  "signatures": [
    {
      "signer": "ITEMBA-ENERGIES (IFMS)",

```

Invalid/expired example:

```json
{ "valid": false }
```

## Safety Rules
- Public endpoint exposes minimal metadata only.
- No file access from this endpoint.
- Invalid/expired tokens return safe `valid=false` response.
- Verification receipt endpoint only returns a generated evidence summary PDF (not original source data).

## Rate Limiting
Verification endpoint is throttled to reduce abuse:
- 30 requests per 60 seconds (per throttler key).

## Source of Truth
Validation is based on `exports` table fields:
- `verification_token`
- `status` must be `ready`
- `expires_at` must be in the future

## Online vs Offline Verification
- **Offline**: PDF signatures can be validated by compliant PDF validators using embedded signature/timestamp/evidence.
- **Online**: verifier checks token status and evidence summary via the public endpoint and receipt download.
